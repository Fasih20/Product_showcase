# ADR-002: Choose SSO Solution for Next.js

Status: accepted

## Context
We need to integrate Single Sign-On (SSO) for Google and Microsoft into our Next.js application. A robust and secure solution is required to handle authentication flows, session management, and provider integration.

## Decision
We will use `NextAuth.js` (now `Auth.js`) as our primary SSO solution. It provides excellent integration with Next.js, supports various OAuth providers, handles session management, and offers a flexible API for custom authentication.

## Consequences
- Dependency on `NextAuth.js` library, requiring regular updates and maintenance.
- Need to configure environment variables for OAuth credentials for each provider.
- Requires creating specific API routes for authentication callbacks (`/api/auth/[...nextauth]/route.ts`).
- Simplifies session management and user data handling, reducing custom implementation effort.
- Potential learning curve for developers unfamiliar with NextAuth.js concepts.
